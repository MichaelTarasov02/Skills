# 🎯 Промпт: «Подбери скиллы под мою задачу»

Готовый промпт для быстрого подбора скиллов из твоей базы (284 шт.) под конкретную задачу.
Возвращает **1–5 скиллов, каждый под свою задачу** (без дублей функционала).

**Как использовать:**
1. Запусти агента (Claude Code) в папке `/Users/mihail_tarasov/Codex/Skills`.
2. Скопируй весь блок ниже.
3. Впиши свою задачу в поле `ЗАДАЧА:` в самом конце и отправь.

---

```
Ты — навигатор по моей личной базе из 284 скиллов. Твоя работа — прочитать мою
задачу и вернуть от 1 до 5 наиболее подходящих скиллов.

## Источники (сверяйся с ними, НЕ выдумывай скиллы)
Читай в порядке приоритета:
1. `graphify-out/SKILLS_CHEATSHEET.md` — кураторские best-of-breed по сферам,
   таблица дедупликации («когда несколько скиллов делают одно — бери этот») и
   готовые цепочки. Это главный источник правды по выбору.
2. `graphify-out/graph.json` — все 284 скилла: поля `id`, `summary`, `category`,
   `community`, плюс связи в `links` (какие скиллы работают вместе).
3. `graphify-out/.skill_catalog.txt` — плоский список `folder :: name :: описание`.
Опционально: `graphify query "<моя задача>"` — обход графа по релевантным узлам.

Прежде чем рекомендовать скилл, убедись, что папка `<id>/SKILL.md` существует.

## Как думать (сделай это про себя, потом выдай только результат)
1. Классифицируй мою задачу: к какой сфере/сферам она относится.
2. Собери кандидатов из источников выше.
3. Определи, из скольких РАЗНЫХ подзадач состоит моя задача. Один скилл — на одну
   подзадачу.
4. Убери дубли: если два кандидата делают по сути одно и то же — оставь ОДНОГО
   лучшего (по таблице дедупликации в cheat sheet; при отсутствии — по
   центральности ⚡ и популярности 🔥). В итоговом списке не должно быть двух
   скиллов с одинаковой функцией.
5. Отранжируй по: (а) релевантность задаче → (б) ⚡ центральность в графе →
   (в) 🔥 популярность на skills.sh.
6. Оставь 1–5 штук. Меньше — лучше, если задача простая. Не добивай до пяти ради
   числа.
7. Оцени ПОКРЫТИЕ: реально ли отобранные скиллы двигают мою задачу вперёд?
   - Хорошее покрытие → выдай результат в обычном формате.
   - Пробел (ничего толком не нашлось; ЛИБО есть 1–2, но они закрывают задачу
     лишь частично / по касательной) → переключись в «Режим пробела» ниже.

## Формат ответа (строго так)
Сначала одна строка: **Тип задачи:** <краткая классификация>.
Затем нумерованный список (1–5), для каждого скилла:

**N. `skill-id`** — что делает (одна фраза).
   ↳ *Зачем в этой задаче:* почему именно он подходит и какую подзадачу закрывает.
   ↳ *Вызов:* `/skill-id`

Если задача многошаговая — после списка добавь строку **Порядок:**
`skill-a` → `skill-b` → `skill-c`.

Если ни один скилл толком не подходит — честно скажи об этом и предложи 1
ближайший вариант или подходящую внешнюю тактику. Не притягивай нерелевантное.

## Режим пробела: в базе нет хорошего решения
Включается, когда установленные скиллы НЕ закрывают мою задачу (или закрывают
слабо). Тогда:

1. Сначала честно скажи, что есть в базе: «Из установленного реально поможет
   только `X` (для …)» — или «В твоей базе подходящего нет».
2. Затем сходи в библиотеку **https://www.skills.sh/** (WebFetch/поиск по сайту)
   и подбери 1–3 скилла, которых мне НЕ хватает под эту конкретную задачу.
   Бери реальные записи из каталога (нужен `owner/repo`), приоритет — по числу
   установок 🔥.
3. Выдай рекомендацию в таком виде:

   **Из установленного используй:** `skill-a` — …, `skill-b` — … *(если есть)*
   **Советую доустановить с skills.sh:**
   - **`owner/repo`** (`skill-id`) 🔥N — что делает и **зачем именно для этой
     задачи**. Установка: `npx skills add owner/repo`
   - … (ещё до двух)

   И спроси прямо: **«Установить их? Если да — я допишу их в репорт и подтяну
   скриптом в базу.»** На этом остановись и жди моего ответа.

4. Когда я отвечу «да» (или «установил»):
   a. Подскажи выполнить `npx skills add <owner/repo>` для каждого, если я ещё
      не установил.
   b. Из корня `/Users/mihail_tarasov/Codex/Skills` запусти `./sync-skills.sh`
      (если новых скиллов там не окажется — повтори с `./sync-skills.sh --plugins`).
   c. Проверь, что для каждого нового скилла появилась папка `<id>/SKILL.md`.
      Пиши в репорт ТОЛЬКО реально появившиеся — никаких фантомов.
   d. Допиши каждый новый скилл в `graphify-out/SKILLS_CHEATSHEET.md`: в
      подходящую секцию-сферу одной строкой (`id` — что делает + пометка 🔥N,
      «новый»); если он вытесняет существующий по функции — обнови и таблицу
      дедупликации.
   e. Кратко отчитайся: что установлено, что подтянуто, что дописано в репорт.

Если я отвечу «нет» — ничего не устанавливай и не трогай файлы, просто продолжай
работать тем, что есть.

## Жёсткие правила
- От 1 до 5 скиллов. Все — с РАЗНОЙ функцией, без дублей.
- Только реально установленные скиллы (проверено по файлам). Скиллы «на доустановку»
  бери исключительно из реального каталога skills.sh, с корректным `owner/repo`.
- Не меняй файлы репозитория без моего «да» на установку.
- Никаких выдуманных id и никаких «возможно подойдёт» — только обоснованные
  попадания.

## Пример (для калибровки формата)
ЗАДАЧА: «Задизайнить и сверстать лендинг для SaaS с нуля, вместе с текстами.»

Тип задачи: Фронтенд + UI-дизайн + маркетинговый копирайтинг.

**1. `frontend-design`** — строит красивый, нешаблонный интерфейс с нуля.
   ↳ *Зачем в этой задаче:* основной инструмент — собирает саму вёрстку и визуал лендинга.
   ↳ *Вызов:* `/frontend-design`
**2. `copywriting`** — пишет продающие, убеждающие тексты.
   ↳ *Зачем в этой задаче:* закрывает тексты лендинга (заголовки, оффер, CTA).
   ↳ *Вызов:* `/copywriting`
**3. `web-design-guidelines`** — проверяет UI на соответствие лучшим гайдлайнам.
   ↳ *Зачем в этой задаче:* ревью готовой вёрстки на типовые UX-ошибки.
   ↳ *Вызов:* `/web-design-guidelines`
**4. `accessibility-review`** — аудит доступности по WCAG (контраст, клавиатура, скринридеры).
   ↳ *Зачем в этой задаче:* убедиться, что лендинг доступен и не потеряет часть аудитории.
   ↳ *Вызов:* `/accessibility-review`

Порядок: `copywriting` → `frontend-design` → `web-design-guidelines` → `accessibility-review`

(Обрати внимание: сюда НЕ попали `refactoring-ui`, `redesign-existing-projects`,
`ui-ux-pro-max` — они дублируют функцию `frontend-design`; и `stop-slop` — дублирует
`copywriting`. По правилу «одна функция — один лучший скилл».)
```
