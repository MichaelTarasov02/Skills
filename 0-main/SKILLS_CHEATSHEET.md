# 🧠 Памятка по скиллам — что и когда использовать

> Персональный справочник по 284 установленным скиллам. Для каждой **сферы** — топ инструментов, по одному лучшему на каждую задачу (без дублей функционала).
> Составлено из двух источников: **граф Graphify** (связи и центральность скиллов в твоей библиотеке) + **популярность на [skills.sh](https://www.skills.sh/)** (реальные счётчики установок).

### Как читать
| Знак | Значение |
|------|----------|
| 🔥 `N` | Установок на skills.sh — насколько скилл популярен в комьюнити |
| ⚡ | **Хаб графа** — скилл с самым большим числом связей в своей области (де-факто «главный») |
| 🔁 | Есть дубли-конкуренты → в блоке «Дедупликация» указан победитель |
| →  | Идёт в связке: этот скилл обычно запускают *перед* следующим |

## 🔁 Главное правило: когда несколько скиллов делают одно и то же — бери этот

Твоя библиотека содержит много перекрывающихся скиллов. Вот победители по спорным задачам:

| Задача | ✅ Бери | ❌ Не трать время на дубли |
|--------|---------|---------------------------|
| Написать тесты через TDD | `tdd` 🔥403K | `test-driven-development` |
| Ревью кода | `code-review` | `code-reviewer`, `review` |
| Отладка по процессу | `systematic-debugging` | `debug`, `diagnosing-bugs` (это для быстрых точечных случаев) |
| Улучшить существующий промпт | `enhance-prompt` | `prompt-optimizer`, `boost-prompt`, `prompt-builder` |
| Стресс-тест плана вопросами | `grilling` | `grill-me`, `grill-with-docs` |
| Прочитать .docx/.xlsx/.pdf как текст | `markitdown` | ручное открытие |
| Ревью договора | `review-contract` | `contract-review` |
| PDF-операции | `pdf` 🔥153K | `pdf-anthropic`, `pdf-processing-pro` |
| Создать документ Word | `docx` 🔥147K | `doc`, `documentation-writer` |
| Сжать общение ради токенов | `caveman` 🔥337K | `caveman-compress`, `caveman-stats` (это утилиты к нему) |

## 1. 🧭 Планирование и мышление (перед любой работой)

*Когда: у тебя есть идея или задача, но ещё не ясно КАК её делать.*

- **`brainstorming`** ⚡ 🔥270K — расширить сырую идею, накидать подходы и развилки. Всегда первым.
- **`grilling`** 🔥511K 🔁 — безжалостно допросить твой план вопросами и найти дыры до старта.
- **`writing-plans`** ⚡ → — превратить спеку в пошаговый план реализации кода.
- **`executing-plans`** — исполнить готовый план с чекпоинтами и ревью между шагами.
- **`decision-mapping`** — разложить сложное решение на варианты с trade-offs.

**Цепочка:** `brainstorming` → `grilling` → `writing-plans` → `executing-plans`

---

## 2. ✍️ Промпт-инжиниринг

*Когда: пишешь или чинишь промпт для LLM.*

- **`prompt-engineering`** ⚡ — спроектировать промпт с нуля (главный, базовый).
- **`enhance-prompt`** 🔁 — улучшить уже существующий промпт (специфичность, структура).
- **`ai-prompt-engineering-safety-review`** — проверить промпт на джейлбрейки и уязвимости.
- **`senior-prompt-engineer`** — системный уровень: RAG, агенты, few-shot, оценка качества.
- **`prompt-engineering-patterns`** — справочник готовых паттернов, когда нужен образец.

---

## 3. 💻 Разработка ПО и архитектура

*Когда: пишешь фичи, проектируешь или чинишь архитектуру.*

- **`senior-frontend`** ⚡ — реализация фронтенд-фич (React/Next/TS). Для фуллстека — `senior-fullstack`, для бэка — `senior-backend`.
- **`architecture`** — ADR: выбрать технологию/паттерн, зафиксировать trade-offs.
- **`codebase-design`** — спроектировать структуру нового кода/модуля с нуля.
- **`improve-codebase-architecture`** 🔥421K — отрефакторить архитектуру существующего кода.
- **`clean-code`** — держать код чистым и без оверинжиниринга (стандарты).

---

## 4. 🐞 Отладка

*Когда: что-то сломалось.*

- **`systematic-debugging`** ⚡ 🔁 — дисциплинированный процесс: воспроизвести → изолировать → первопричина. Для непонятных багов.
- **`debug`** — интерактивная сессия по конкретной ошибке/стектрейсу («работает в staging, не в prod»).
- **`find-bugs`** ⚡ — прочесать изменения ветки на баги и уязвимости проактивно.

---

## 5. ✅ Тесты и QA

*Когда: нужно покрытие или уверенность, что сделано верно.*

- **`tdd`** 🔥403K 🔁 — писать код через тесты (red-green-refactor). Строгий процесс.
- **`verification-before-completion`** ⚡ — проверить, что задача реально доведена до конца, перед тем как сказать «готово».
- **`senior-qa`** ⚡ — стратегия тестирования и тест-планы.
- **`webapp-testing`** — прогнать веб-приложение end-to-end в браузере.

---

## 6. 🔍 Ревью кода

*Когда: свой или чужой код нужно проверить.*

- **`code-review`** ⚡ 🔁 — найти баги, уязвимости и проблемы качества в изменениях (главный).
- **`requesting-code-review`** → **`receiving-code-review`** — как грамотно запросить ревью и как принять правки. Пара для командной работы.

---

## 7. 🎨 Фронтенд и UI-дизайн

*Когда: делаешь или улучшаешь интерфейс.*

- **`frontend-design`** ⚡ 🔥647K — построить красивый, нешаблонный UI с нуля (самый популярный дизайн-скилл вообще).
- **`refactoring-ui`** — прокачать уже существующий интерфейс; для полного апгрейда проекта — `redesign-existing-projects`.
- **`accessibility-review`** — аудит доступности (WCAG: контраст, клавиатура, скринридеры).
- **`web-design-guidelines`** 🔥452K — проверить UI на соответствие лучшим гайдлайнам.
- **`apple-design`** — «эппловский» вкус: жесты, пружинная анимация, глубина, типографика.

> Хочешь один мега-скилл вместо пяти? `impeccable` или `ui-ux-pro-max` — широкие «всё-в-одном» по UI/UX.

---

## 8. 🧩 Дизайн-системы и токены

*Когда: строишь систему, а не отдельный экран.*

- **`ui-design-system`** ⚡ — токены, документация компонентов, handoff разработчикам.
- **`design-system-patterns`** — паттерны построения дизайн-системы.
- **`theme-factory`** — генерация тем и цветовых палитр.
- **`better-icons`** — иконки под систему.

## 9. 📱 Мобильная разработка (Flutter)

*Когда: делаешь мобильное приложение на Flutter.*

- **`flutter-expert`** ⚡ (хаб на 26 связей — центр всего Flutter) — точка входа, роутит в 20+ узких Flutter-скиллов.
- **`flutter-architecting-apps`** — архитектура приложения.
- **`flutter-managing-state`** — управление состоянием.
- **`mobile-design`** — мобильный UX и паттерны.

> Есть узкие: `flutter-building-forms`, `flutter-animating-apps`, `flutter-testing-apps`, `flutter-theming-apps` и др. — заходи через `flutter-expert`.

## 10. 🤖 Агенты, скиллы и оркестрация

*Когда: строишь агентов или сами скиллы.*

- **`find-skills`** 🔥2.4M — найти нужный скилл под задачу (самый устанавливаемый скилл в мире).
- **`skill-creator`** 🔥308K — создать новый скилл.
- **`writing-skills`** — как писать хорошие скиллы (лучшие практики).
- **`agent-development`** ⚡ — создать субагента.
- **`dispatching-parallel-agents`** — запустить несколько агентов параллельно; для последовательной разработки — `subagent-driven-development`.

*Когда: работаешь с числами и датасетами.*

- **`data-visualization`** ⚡ — построить графики; для дашборда — `build-dashboard`.

## 12. 📄 Документы, файлы и медиа

*Когда: генеришь или читаешь файлы.*

- **`docx`** 🔥147K / **`xlsx`** 🔥130K / **`pdf`** 🔥153K — создать/редактировать Word / Excel / PDF (по формату).
- **`canvas-design`** — визуальные макеты и постеры.
- **`ai-image-generation`** — сгенерировать картинки; лого — `logo-creator`.

## 🔗 Готовые сквозные цепочки (по связям графа)

Граф показал типичные последовательности «precedes» — используй их как рецепты:

1. **Фича с нуля:** `brainstorming` → `grilling` → `write-spec` → `writing-plans` → `executing-plans` → `code-review` → `verification-before-completion`
2. **Продукт:** `product-brainstorming` → `to-prd` → `to-issues` → (разработка) → `metrics-review`
3. **Контент/SEO:** `content-strategy` → `content-creation` → `stop-slop` → `seo-audit`
4. **Стартап:** `startup-ideation` → `startup-validator` → `market-sizing-analysis` → `pricing-strategy` → `startup-financial-modeling`
5. **Бренд-контент:** `discover-brand` → `guideline-generation` → `brand-voice-enforcement`
6. **Найм:** `resume-tailor` → `resume-bullet-writer` → `cover-letter-generator` → `linkedin-profile-optimizer`
