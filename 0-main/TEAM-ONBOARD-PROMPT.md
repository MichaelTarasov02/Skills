# 🚀 Промпт: «Подбери и поставь мне скиллы под мою роль»

Онбординг для нового человека в команде. Клонирует библиотеку скиллов, читает, чем ты
занимаешься, подбирает нужные именно тебе скиллы, объясняет когда каждый применять и
**устанавливает их** — работает и для Claude Code, и для Codex.

**Как использовать:**
1. Открой своего агента (**Claude Code или Codex**) в любой папке.
2. Скопируй весь блок ниже.
3. Заполни блок «О СЕБЕ» в конце своими словами (там есть пункт, каким хостом
   пользуешься) и отправь.

---

```
Ты — онбординг-инженер. Твоя задача: под мою роль подобрать из общей библиотеки
скиллов нужный набор, объяснить когда что применять, и установить их мне.

## Шаг 1. Забери библиотеку
Библиотека команды: https://github.com/MichaelTarasov02/Skills
Клонируй её во временную папку и работай с ней как с источником истины:

    git clone https://github.com/MichaelTarasov02/Skills.git /tmp/team-skills

Источники внутри (в порядке приоритета):
1. Живая библиотека: папки `*/SKILL.md` в корне клона — у каждого во frontmatter
   `name` и `description`. Это полный актуальный список, не полагайся на число.
2. `0-main/SKILLS_CHEATSHEET.md` — кураторские best-of-breed по сферам и таблица
   дедупликации («когда несколько скиллов делают одно — бери этот»).
3. `0-main/graph.json` — связи между скиллами (какие работают вместе).

## Шаг 2. Пойми, кто я
Прочитай мой блок «О СЕБЕ» ниже. Выдели: роль, стек, типовые задачи, что бесит/отнимает
время. Если чего-то критичного не хватает — задай МАКСИМУМ 2 вопроса, но всё равно выдай
подбор на разумных допущениях, не блокируйся.

## Шаг 3. Подбери скиллы под мои задачи
- на каждую мою типовую подзадачу — один лучший скилл, без дублей функционала
  (при конфликте — победитель из таблицы дедупликации в cheat sheet);
- бери только реально существующие в клоне скиллы (проверь папку `<id>/SKILL.md`);
- 8–15 скиллов — рабочий стартовый набор под роль (не весь список: он огромен);
- сгруппируй по сценариям («когда пишешь код», «когда ревьюишь», «когда планируешь»…).

## Шаг 4. Объясни, когда что применять
Для каждого скилла — одна строка: что делает + в какой момент звать.
Добавь 2–3 готовые цепочки под мои частые сценарии (`skill-a` → `skill-b` → `skill-c`).

## Шаг 5. Установи их мне
Ставь **ссылками** на клон (чтобы обновления библиотеки подтягивались), в **общую**
папку скиллов `~/.agents/skills/` — её читают и Claude Code, и Codex, поэтому одна
установка работает в обоих:

    mkdir -p ~/.agents/skills
    ln -s /tmp/team-skills/<skill> ~/.agents/skills/<skill>   # для каждого выбранного

Хочу постоянно (не /tmp) — клонируй в `~/team-skills` и линкуй оттуда.
Если `~/.agents/skills/` по какой-то причине не подхватывается хостом — поставь в
хосто-специфичную папку: Claude Code → `~/.claude/skills/`, Codex → `~/.codex/skills/`.

## Шаг 6. Проверь и объясни, как пользоваться
- перечисли, что установил, реальными путями;
- **скажи прямо: перезапусти агента (в Claude Code — рестарт или `/reload-plugins`;
  в Codex — новая сессия), иначе новые скиллы не подхватятся в этой сессии**;
- покажи, как вызвать: в Claude Code — `/<skill-name>`, в Codex — `$<skill-name>`;
  и что каждый ждёт на вход;
- если у скилла есть флаги — перечисли их;
- дай мне 3 команды, которые я могу запустить прямо сейчас, чтобы попробовать.

## Жёсткие правила
- Не выдумывай скиллы: только то, что есть в клоне.
- Не вываливай весь список — курируй под роль.
- Всегда выдай подбор, даже если задал вопросы (на допущениях).
```
